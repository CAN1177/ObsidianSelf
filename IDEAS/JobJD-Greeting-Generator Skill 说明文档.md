# JobJD-Greeting-Generator Skill 说明文档

> 面向 BOSS 直聘等国内招聘软件，根据岗位 JD 与候选人信息自动生成高回复率打招呼话术。[web:31][web:32]

---

## 1. Skill 基本信息

### 1.1 名称

- `JobJD-Greeting-Generator`

### 1.2 目标

- 根据指定岗位 JD 与候选人信息，自动生成：
  - 首次打招呼话术（≤100 字）
  - “已读未回 / 想再跟进一次”话术（≤120 字）
- 适配 BOSS 直聘、智联、前程无忧等国内招聘 App 的 IM 对话场景。[web:31][web:37]

### 1.3 适用场景

- 求职者在投递/感兴趣职位时，替换系统默认招呼语。
- 已经被 HR 看过/已读但未回复，想礼貌跟进一次。
- 想批量、快速生成个性化招呼内容的工具/插件场景。[web:31][web:37]

---

## 2. 输入参数设计

```text
必填：
- jd_text:        文本版岗位 JD（可由网页解析或用户粘贴）
- position_name:  岗位名称，例如「后端开发工程师」
- company_name:   公司名称
- candidate_brief: 候选人一句话简介（学校/公司 + 职位/方向 + 年限 + 当前状态）

选填：
- candidate_highlights: 2–5 条与 JD 高度相关的技能/项目/成果要点
- candidate_status:     如「在职-考虑机会」「已离职-可随时到岗」
                         「大四-每周可实习3天」等
- hr_role:              HR 身份，如「HR」「技术负责人」「业务负责人」
- hr_gender:            HR 性别（male/female/unknown），仅用于称呼细节
- tone_style:           语气风格，如「正式」「自然专业」「略活泼」
- scenario:             场景：
                        - "first_contact"（首次打招呼）
                        - "follow_up_read"（已读未回跟进）
                        - "follow_up_silent"（投递后无任何反馈）
- length_limit:         字数上限，默认 100 字
- platform:             平台，如 "BOSS直聘" / "智联招聘"
```

---

## 3. 输出格式规范（Markdown）

统一输出为 Markdown 段落，方便直接复制到聊天框使用或二次展示。

```markdown
### 打招呼话术

> 这里是一段最终招呼语，不超过 length_limit 字。

### 结构说明（可选，用于调试或教学）

- 身份标签：...
- 岗位匹配点：...
- 礼貌收尾与轻邀请：...
- 场景：first_contact / follow_up_read / follow_up_silent
```

在面向终端用户时，可仅返回「打招呼话术」部分。

---

## 4. 话术结构规则（核心套路）

整体采用“三段式结构”：**身份标签 → 岗位匹配点 → 礼貌收尾 + 轻邀请**。[web:21][web:29]

### 4.1 身份标签（Who + 做什么 + 多久）

**目标**：一句话让 HR 立刻知道你是谁、做什么、经验大致怎样。[web:21]

- 信息来源：
  - `candidate_brief`
  - `candidate_status`
- 需覆盖要素：
  - 学校/公司：当前或最近的学校/公司
  - 职位/方向：岗位 + 方向（如「后端开发」「用户运营」）
  - 年限：实习 / X 年经验 / 应届生
  - 当前状态：在职/离职/可实习时间/目标城市

**生成模板示例**

```text
「您好，我是【学校/公司】的【职位/方向】，【X 年/应届】经验，
目前处于【在职-考虑机会/已离职/寻找实习】阶段。」
```

### 4.2 岗位匹配点（与 JD 的契合）

**目标**：用 1–2 句点出 2–3 个与 JD 高度相关的技能/项目/成果，而不是空话。[web:21][web:26][web:29]

- 从 `jd_text` 中抽取：
  - 岗位职责要点：前 3–5 条
  - 技能要求要点：前 3–7 个技能/工具/行业名
- 从 `candidate_highlights` 中筛选与上述关键词高度重合的 2–3 条，优先：
  - 真实项目/业绩描述（带数字或结果）
  - 与 JD “加分项”对应的经历

**生成模板示例**

```text
「主要负责过【与你 JD 相似的项目】，熟悉【核心技能1、核心技能2】，
最近在【项目/岗位】中取得了【可量化的小成果】。」
```

### 4.3 礼貌收尾 + 轻邀请

**目标**：表达“希望沟通”的意愿，但不给 HR 压力，避免命令式或过度卑微的说法。[web:20][web:29]

**生成模板示例**

```text
「看到您发布的『【position_name】』岗位感觉很匹配，
如方便的话，希望能占用您几分钟，简单了解一下具体要求。
感谢查看，祝工作顺利。」
```

### 4.4 字数与内容优先级控制

- 默认总字数 ≤ `length_limit`（默认 100）。
- 拼接三段内容后，如超过上限：
  - 先删除重复信息（如公司名重复出现）。
  - 再缩减修饰词和次要成果细节。
  - 始终保留：身份标签 + 至少 2 个核心匹配点关键词。

## 5. 生成流程（算法步骤）

### 5.1 输入解析

1. 接收 `jd_text`（必要时由网页解析获得 JD 正文）。
2. 对 JD 进行关键词提取：
   - 职责要点：提炼前 3–5 条职责描述。
   - 技能要点：提炼前 3–7 个技能/工具/行业名。
3. 基于词汇特征将关键词标记为“必备”或“加分项”（例如包含“必须”“优先”“加分”等词）。[web:21][web:29]

### 5.2 生成候选人画像与亮点匹配

1. 根据 `candidate_brief` + `candidate_status` 生成标准化的“身份标签”句子。
2. 将 `candidate_highlights` 与 JD 关键词计算匹配度：
   - 完全相同的关键词匹配度最高。
   - 近义词、同一技术栈或同一业务场景次之。
3. 按匹配得分排序，选出 2–3 条亮点用于生成“岗位匹配点”部分。

### 5.3 场景化话术生成

#### 5.3.1 场景 A：首次打招呼（`first_contact`）

1. 生成 1 句“身份标签”。
2. 生成 1–2 句“岗位匹配点”，至少包含 2 个 JD 核心技能或核心职责，并尽量带 1 个可量化结果。
3. 生成 1 句“礼貌收尾 + 轻邀请”。
4. 三部分合并后根据 `length_limit` 做截断与精简。

#### 5.3.2 场景 B：已读未回跟进（`follow_up_read`）

遵循“承认之前打扰 + 给对方台阶 + 补充信息 + 再次轻邀请”的结构。[web:10][web:21]

生成逻辑示例：

- 开头：承接前文  
  - 「XX 经理您好，之前就『【position_name】』给您发过一条信息，不确定是否方便查看。」
- 补充价值或信息  
  - 「如果您对我的背景还有疑虑，我可以补充更详细的项目说明/作品集。」
- 给出选择与台阶  
  - 「若当前阶段不太匹配，也麻烦您简单告知，我会据此调整方向。」
- 收尾  
  - 「再次感谢您的时间，祝工作顺利。」

#### 5.3.3 场景 C：投递后完全无反馈（`follow_up_silent`）

结构类似 B，但不提“已读”：

- 「您好，我是之前投递『【position_name】』的【候选人身份标签】。」
- 「想确认一下您是否已经完成这一批次的初筛，如需补充信息我可以随时提供。」
- 「若目前暂无合适机会，也欢迎告知一声，方便我合理规划后续求职，谢谢您。」

### 5.4 语气风格控制（`tone_style`）

- `正式`：使用「您好」「贵司」「方便的话」等较正式表达，减少口语。
- `自然专业`（默认）：介于正式与口语之间，适合大部分互联网岗位。[web:21]
- `略活泼`：在不影响专业度前提下，少量使用「刚好」「挺感兴趣」等口语化词汇，但避免表情符号。

---

## 6. 安全与限制

- **不捏造经历**：仅基于 `candidate_highlights` 中的真实信息进行重组，不虚构项目或数据。
- **避免攻击与施压**：禁止生成「必须尽快回复」「为什么不回我」等具有施压或指责意味的语句。[web:20][web:29]
- **隐私控制**：除非用户在输入中主动提供，否则不要生成身份证号、手机号等敏感信息。
- **平台约束**：在 `platform` 为特定平台时，遵守该平台文字规范（如禁止带联系方式等）。

## 7. 调用示例（逻辑说明）

```text
1. 前端表单收集：
   - 岗位 JD（粘贴或 URL）
   - 个人一句话简介
   - 3–5 条项目/技能亮点
   - 场景（首次/跟进）
   - 语气风格（可选）

2. 后端调用 JobJD-Greeting-Generator：
   - 按上文输入参数组装请求
   - 由大模型按「身份标签 → 匹配点 → 收尾」结构生成话术

3. 前端展示：
   - 以 Markdown 形式展示结果
   - 支持一键复制到剪贴板
   - 可选：同时展示“结构说明”，帮助用户理解和调整
```
